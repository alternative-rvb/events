# Page responsive HTML et CSS - Évènements à venir

## Compétences visées

### Réaliser des interfaces utilisateur statiques web ou web mobile

- **Compétences** : Développement de pages web en utilisant HTML5 et CSS3, compréhension de la mise en page responsive.
- **Exemple** : Codage en HTML5 et CSS3 pour structurer des pages web et appliquer des styles adaptatifs sur différents supports (desktop, tablette, mobile).

## Introduction

Bienvenue à votre première journée de formation en HTML et CSS. Aujourd'hui, nous allons plonger dans le monde du développement web en commençant par les bases.

Ne vous inquiétez pas si vous ne terminez pas l'exercice ou si le résultat n'est pas une copie conforme de l'exemple ; l'objectif est de vous familiariser avec les balises HTML principales et d'initier l'application de styles CSS.

## Objectifs

Créer un article présentant présentant un événement à venir et intégrer une carte Google Maps pour présenter le lieu de l'événement.

### Thématique libre

Vous pouvez choisir le thème de votre événement parmi :

- **Sport** (ski, marathon, tournoi, etc.)
- **Jeux vidéo** (compétition e-sport, salon, lancement)
- **Culture** (festival, exposition, concert)
- **Technologie** (conférence, hackathon, salon)
- Ou tout autre événement de votre choix

⚠️ **IMPORTANT - Structure de la page** :

- La **structure HTML** de votre page doit être **identique** aux captures d'écran fournies
- Seuls les **contenus textuels** et les **images** peuvent être adaptés à votre thématique
- Respectez l'**ordre et l'organisation** des sections (header, navigation, article, footer)
- Les **éléments de mise en page** (nombre de sections, disposition) doivent être conservés

L'article comprendra notamment :

- Les informations détaillées sur l'événement
- Une carte interactive Google Maps (iframe) indiquant le lieu de l'événement, positionnée après la section "Lieu"
- Une mise en page responsive et attractive

## Instructions

### 1. Structure HTML de base

- Créez la structure de base de votre document HTML (`<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`)
- Ajoutez un `<title>` à votre page
- Intégrez les balises `<meta>` nécessaires (charset UTF-8, viewport)
- Organisez votre code avec des commentaires
- Utilisez les balises sémantiques : `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`

### 2. En-tête du site `<header>`

- Créez une navigation (`<nav>`) avec le logo et des liens vers : Accueil, Classement, Événements, Contact

### 3. Contenu principal `<main>`

- Structurez l'article avec une balise `<article>` comprenant :
  - Une image principale de l'événement
  - Le titre de l'article (`<h1>`)
  - Une section méta (date, catégorie, auteur, réseaux sociaux)
  - Plusieurs paragraphes avec des sous-titres (`<h2>`, `<h3>`)
  - Une section "Lieu"
  - Un iframe Google Maps (positionné après la section "Lieu")
  - Un bouton de réservation

### 4. Pied de page `<footer>`

- Créez trois sections : À propos, Médias sociaux, Classement

### 5. Stylisation CSS

Créez un fichier CSS externe (`css/style.css`) :

- **Commencez par les sélecteurs de balises** : `body`, `header`, `nav`, `h1`, `p`, `img`, `footer`, etc.
- Ajoutez des classes CSS progressivement si nécessaire
- Choisissez une palette de couleurs harmonieuse
- Appliquez des effets au survol (`:hover`) sur les liens et boutons

### 6. Responsive Design

- Le site doit s'adapter aux résolutions :
  - **Desktop** : 1920x1080 pixels
  - **Mobile** : 360x800 pixels
- Utilisez les media queries pour adapter votre mise en page
- Rendez les images responsive `width: 100%`, `height: auto` ou mieux encore `max-width: 1200px` par exemple pour le contenu principal

### 7. Validation et tests

- Validez votre code : [HTML Validator](https://validator.w3.org/) | [CSS Validator](https://jigsaw.w3.org/css-validator/)
- Testez sur différents navigateurs et tailles d'écran

## Conseils et bonnes pratiques

- Commencez par la structure HTML avant le CSS
- Privilégiez les sélecteurs de balises (`body`, `header`, `nav`, `h1`, `p`, `img`, `footer`)
- Introduisez les classes CSS uniquement si nécessaire
- Respectez la structure des captures d'écran (seuls textes et images peuvent varier)
- Testez régulièrement avec les outils de développement (F12)
- Commentez votre code
- L'erreur fait partie de l'apprentissage, expérimentez !
