# Page responsive HTML - CSS - Grille d'évènements à venir

## Compétences visées

### Réaliser des interfaces utilisateur statiques web ou web mobile

- **Compétences** : Développement de pages web en utilisant HTML5 et CSS3, compréhension de la mise en page responsive.
- **Exemple** : Codage en HTML5 et CSS3 pour structurer des pages web et appliquer des styles adaptatifs sur différents supports (desktop, tablette, mobile).

## Introduction

Bienvenue à la suite de votre formation en HTML et CSS dans le cadre du programme "Concepteur Designer UI".

Après avoir exploré les bases, nous allons désormais approfondir votre compréhension et vos compétences en Responsive Web Design.

## Objectif

L'objectif est de créer une nouvelle page web responsive pour présenter une grille d'évènements à venir.

Votre mission est de reproduire aussi fidèlement que possible les captures d'écrans fournie dans la maquette.

⚠️ **IMPORTANT - Structure de la page** :

- La **structure HTML** de votre page doit être **identique** aux captures d'écran fournies
- Seuls les **contenus textuels** et les **images** peuvent être adaptés à votre thématique
- Respectez l'**ordre et l'organisation** des sections (header, navigation, article, footer)
- Les **éléments de mise en page** (nombre de sections, disposition) doivent être conservés

## Instructions

### 1. **Amélioration de la structure HTML :**

- Organisez proprement votre code HTML. Les commentaires peuvent vous aider à séparer clairement les sections de la page.
- Rendez votre site plus accessible en utilisant les balises sémantiques (`<header>`, `<main>`, `<footer>`), etc.

### 2. **Responsive Design :**

- Utilisez `max-width` et `margin: auto;` pour les conteneurs principaux afin qu'ils s'ajustent aux différentes largeurs.
- Implémentez des `media queries` avec la propriété `columns`, `flex` ou `grid` dans votre CSS pour adapter le site aux différentes tailles d'écran.
  - Mobile: 360x800 - 1 colonne
  - Desktop: 1920x1080 - 2 colonnes
- Rendez les images responsive avec `width: 100%;` et `height: auto;`.

### 3. **Référence visuelle :**

- Vous devez essayer de faire correspondre votre travail le plus fidèlement possible avec la capture d'écran fournie. Faites attention aux détails tels que la taille des textes, les couleurs, et la mise en page.

### 4. **Validation et test :**

- Utilisez des outils de validation en ligne pour vérifier l'absence d'erreurs dans votre code HTML et CSS (w3 Validator).
- Testez le rendu de votre site sur différents appareils et navigateurs pour garantir une compatibilité optimale.
